# Architecture
-   Does the class have one responsibility?
-   Does it follow the approved ADRs?
-   Is business logic in the correct place?
# Validation
-   Can invalid objects be created?
-   Are all business rules enforced?
# Documentation
-   Module docstring present?
-   Class docstring present?
-   Public methods documented?
# Naming
-   Meaningful class names?
-   Meaningful method names?
-   Consistent naming?
# Testing
-   Happy path tested?
-   Invalid input tested?
-   Edge cases tested?
# Git
-   Meaningful commit message?
-   Documentation updated if required?
